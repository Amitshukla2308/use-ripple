## 2025-02-21 - Command Injection in Slash Commands
**Vulnerability:** The `apps/cli/commands` module implementations (such as `_git`, test runners, linters, and mcp management) directly mapped unvalidated user inputs and system utilities to shell execution by passing `shell=True` to `subprocess.run()`. This allowed for arbitrary command injection if malicious payloads were supplied to the slash commands.
**Learning:** Python's `subprocess.run(..., shell=True)` executes via the system shell, which interprets metacharacters like `;`, `&&`, and `|`. Using `shlex.split()` correctly tokenizes the input into a list of arguments, avoiding the system shell evaluation completely. Additionally, using shell execution simply to check for command existence via `which <cmd>` is inefficient and insecure.
**Prevention:** In the `apps/cli/commands` package, any `subprocess` calls must use array-based arguments (`shell=False`, which is the default when passing a list). Always use `shlex.split()` to safely parse user inputs meant for the command line, and use standard library utilities like `shutil.which()` instead of shelling out to determine if a binary exists.
